# Docker Desktop Module 1

Welcome to the first module of the Docker Desktop learning path! In this module, you will learn about Docker Desktop and its basic features.

## Learning objectives

By the end of this module, you should be able to:

- Understand what Docker Desktop is and its purpose.
- Install Docker Desktop on your computer.
- Learn how to use Docker Desktop to manage containers.
- Familiarize yourself with the Docker Desktop interface and its components.

## Chapter 1: Introduction to Docker Desktop

### 1.1 What is Docker Desktop?

Docker Desktop is a lightweight, user-friendly, and powerful desktop application that provides a convenient way to run Docker containers on your local machine. It is designed to work alongside Docker Engine, which is the underlying technology that powers Docker containers.

### 1.2 Installing Docker Desktop

To install Docker Desktop, follow these steps:

1. Visit the [Docker Desktop download page](https://www.docker.com/products/docker-desktop).
2. Download the appropriate installer for your operating system (Windows, macOS, or Linux).
3. Run the installer and follow the on-screen instructions to complete the installation.

### 1.3 Using Docker Desktop to manage containers

Once Docker Desktop is installed, you can use it to manage Docker containers on your local machine. Here are some basic steps to get started:

1. Open Docker Desktop.
2. Click on the "Docker" menu in the menu bar and select "Preferences" to open the preferences window.
3. In the preferences window, you can configure various settings, such as resource allocation, networking, and security options.
4. To run a container, click on the "+" button in the Docker Desktop window and select the container image you want to run.
5. Once the container is running, you can access its terminal, view logs, and manage its resources using the Docker Desktop interface.

### 1.4 Familiarizing yourself with the Docker Desktop interface

Docker Desktop provides a user-friendly interface that allows you to manage your Docker containers and interact with Docker Engine. Here are some key components of the interface:

- **Dashboard**: The dashboard is the main screen that shows you an overview of your running containers, images, and other Docker resources.
- **Containers**: The containers tab lists all the running containers on your local machine. You can view logs, access the container terminal, and manage container resources from this tab.
- **Images**: The images tab lists all the Docker images available on your local machine. You can pull new images, delete existing images, and build new images from Dockerfiles.
- **Volumes**: The volumes tab lists all the Docker volumes on your local machine. You can manage volumes, such as creating, deleting, and mounting volumes.
- **Settings**: The settings tab allows you to configure various Docker settings, such as resource allocation, networking, and security options.
